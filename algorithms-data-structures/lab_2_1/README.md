## Рівень 3 — Диференціальне рівняння

**Файл:** `./lab2/level3.js`

**Завдання:** dy/dx = cos(x) − y,  y(0) = 1,  x ∈ [0, 2],  h = 0.2

**Метод Рунге-Кутта 2-го порядку (Хьюн):**

```
k1    = h · f(xₙ, yₙ)
k2    = h · f(xₙ + h, yₙ + k1)
yₙ₊₁ = yₙ + (k1 + k2) / 2
```

**Точний розв'язок:**  
`y(x) = (sin(x) + cos(x)) / 2 + C·e^(−x)`,  де `C = (y₀ − 1/2)·eˣ⁰`

**Ключові рядки:**

| Рядок | Код | Що робить |
|-------|-----|-----------|
| 14 | `function f(x, y)` | Права частина ДР: cos(x) − y |
| 20 | `function exactSolution(x, x0, y0)` | Аналітичний розв'язок для порівняння |
| 35–37 | `k1, k2, yNext` | Одна ітерація RK2 |
| 38 | `if (x + h > xEnd) h = xEnd - x` | Підрізка останнього кроку до xEnd |
| 44 | `error = abs(yNext - yExact)` | Порівняння з точним розв'язком |

**Порядок точності:** O(h²) — на порядок краще за метод Ейлера O(h¹)

## Відповіді на контрольні питання

**1. Призначення і класифікація числових методів**  
Числові методи — алгоритми наближеного розв'язання задач, які не мають аналітичного розв'язку. Класифікують: за задачею (інтегрування, диференціювання, алгебричні рівняння, ДР), за порядком точності O(hⁿ), за стійкістю.

**2. Метод трапецій**  
Крива між вузлами апроксимується відрізком прямої (трапеція). Графічно — замість плавної кривої рисуємо ламану і рахуємо площу трапецій під нею. Похибка O(h²).

**3. Метод прямокутників**  
Площа під кривою апроксимується прямокутниками. Лівих/правих (O(h¹)) — висота за значенням на краю. Середніх (O(h²)) — висота за значенням у середині відрізка.

**4. Метод Сімпсона**  
Між трьома сусідніми вузлами проводиться парабола. Похибка O(h⁴) — найточніший з трьох. Вимагає парну кількість відрізків.

**5. Метод бісекції**  
Якщо f(a)·f(b) < 0 — корінь між a і b (теорема Больцано). Ділимо навпіл: якщо f(a)·f(mid) < 0 — корінь у лівій половині, інакше — у правій. Гарантована збіжність за O(log₂(1/ε)) ітерацій.

**6. Метод дотичних (Ньютона)**  
З точки x₀ проводимо дотичну до графіка, її перетин з OX — нове наближення xₙ₊₁ = xₙ − f/f'. Квадратична збіжність (кількість вірних цифр подвоюється). Умова: f·f'' > 0 на початковій точці.

**7. Метод хорд**  
Проводимо хорду між (a, f(a)) і (b, f(b)). Перетин з OX: x = a − f(a)·(b−a)/(f(b)−f(a)). Замінюємо кінець з тим самим знаком. Надлінійна збіжність.

**8. Метод Ейлера**  
yₙ₊₁ = yₙ + h·f(xₙ, yₙ). Рухаємось по дотичній на кожному кроці. Похибка O(h¹). Простий, але неточний — Рунге-Кутта точніший.

**9. Рунге-Кутта 2-го порядку**  
k1 = h·f(xₙ, yₙ), k2 = h·f(xₙ+h, yₙ+k1), yₙ₊₁ = yₙ + (k1+k2)/2. Усереднює нахил на початку і в кінці кроку. Похибка O(h²).

**10–11. Рунге-Кутта 3-го та 4-го порядку**  
Використовують 3 або 4 проміжні точки для обчислення середньозваженого нахилу. RK4 — стандарт: 4 обчислення f на крок, похибка O(h⁴). Практичний компроміс між точністю і вартістю обчислень.
